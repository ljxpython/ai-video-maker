# 环境部署提示词

更新时间：2026-06-09

把下面这段话直接发给支持代码执行的 AI，就可以让它按 `uv` 方式完成本仓库的环境部署、skills 入口检查和基础验证。

```text
请帮我在当前仓库里完成 AI Video Maker 的环境部署。

要求：
1. 先检查 uv 是否可用；如果不可用，按官方方式安装 uv。
2. 官方安装方式优先使用：curl -LsSf https://astral.sh/uv/install.sh | sh
3. 如果本机没有 Python 3.12，就执行 `uv python install 3.12`，然后 `uv python pin 3.12`。
4. 进入仓库根目录，执行 uv sync，创建或更新 .venv。
5. 确认仓库根目录下已经有 .agents/skills，并且它能指向 skills 目录；如果缺失，请补上这个入口。
6. 用 uv run 依次执行：
   - ai-video-maker skills validate
   - ai-video-maker validate --pipeline pipeline.example.yml
   - python -m unittest discover -s tests
7. 如果需要浏览器录制，再执行 uv run playwright install chromium。
8. 不要读取任何 token、cookie、密码或私有配置。
9. 完成后，把部署结果、已安装内容和下一步建议告诉我。
```

如果你只想手动执行，最小命令是：

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
uv sync
uv run ai-video-maker skills validate
uv run ai-video-maker validate --pipeline pipeline.example.yml
uv run python -m unittest discover -s tests
```
