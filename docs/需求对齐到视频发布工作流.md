# 需求对齐到视频发布工作流

更新时间：2026-06-08

## 1. 核心想法

项目目标是做一个 AI 视频制作助手。用户只需要说一个需求，AI 负责理解、追问、拆解、规划、制作和交付。

标准流程：

```text
用户提出需求
-> AI 理解意图并补全假设
-> AI 给出视频方案
-> 用户确认方案
-> AI 给出详细执行步骤
-> 用户确认执行
-> AI 调用浏览器/桌面/剪辑/配音/字幕工具制作视频
-> AI 自动质检
-> AI 生成发布包
-> 用户确认后发布 YouTube
```

这个流程的关键不是“AI 直接瞎干”，而是分阶段对齐。每个阶段都有明确产物和确认点。

## 2. 分阶段确认协议

### 2.1 Gate 1：需求确认

AI 需要把用户的模糊需求整理成 brief。

输入示例：

```text
帮我做一个视频，介绍这个工具怎么用。
```

AI 输出：

```yaml
goal: "制作一个工具使用演示视频"
audience: "对该工具感兴趣的技术用户"
platform: "youtube"
duration: "3-5 分钟"
style: "技术讲解，AI 配音，字幕清晰"
source_material:
  - "用户提供的仓库、网址、本地软件或操作步骤"
assumptions:
  - "如果用户未指定平台，默认 YouTube 横屏"
  - "如果用户未指定语言，默认中文"
needs_user_confirmation: true
```

确认后才能进入视频方案。

### 2.2 Gate 2：视频方案确认

AI 输出视频结构、受众、节奏、素材需求和风险。

输出示例：

```yaml
video_plan:
  title_working: "用 AI 自动完成技术视频制作"
  aspect_ratio: "16:9"
  target_duration: 240
  sections:
    - id: "hook"
      duration: 15
      goal: "展示最终效果"
    - id: "problem"
      duration: 30
      goal: "解释用户痛点"
    - id: "workflow"
      duration: 90
      goal: "演示核心流程"
    - id: "demo"
      duration: 90
      goal: "桌面或浏览器实操"
    - id: "summary"
      duration: 30
      goal: "总结价值和下一步"
required_assets:
  - type: "browser_recording"
  - type: "desktop_recording"
  - type: "narration"
  - type: "subtitles"
risks:
  - "YouTube 上传需要用户确认"
  - "Chrome 登录态只用于用户确认的目标站点"
```

确认后才能进入详细执行步骤。

### 2.3 Gate 3：执行步骤确认

AI 输出详细执行计划。

执行计划必须包括：

- 要读取哪些资料。
- 要打开哪些网页或软件。
- 要录制哪些操作。
- 要生成哪些脚本和字幕。
- 要输出哪些视频版本。
- 哪些动作需要用户中途介入。
- 哪些动作禁止自动执行。

示例：

```yaml
execution_plan:
  steps:
    - id: "read_source"
      tool: "filesystem"
      action: "读取需求文档和素材"
      side_effect: false
    - id: "record_demo"
      tool: "$browser"
      action: "录制网页操作演示"
      side_effect: false
    - id: "operate_desktop"
      tool: "$computer-use"
      action: "操作本地 GUI 软件录制素材"
      side_effect: "可能改变本地软件状态"
      confirmation: "when_needed"
    - id: "upload_youtube"
      tool: "$chrome"
      action: "打开 YouTube Studio 并准备上传"
      side_effect: "上传文件到第三方平台"
      confirmation: "required"
```

确认后才能开始实操制作。

### 2.4 Gate 4：发布确认

视频制作完成后，AI 可以准备发布包，但不能直接发布。

发布包包括：

```text
video.mp4
thumbnail.png
title.txt
description.md
tags.txt
upload_checklist.md
```

发布前必须确认：

```text
确认上传平台、账号、文件、标题、简介、可见性和是否立即发布。
```

## 3. GUI 工具边界

### 3.1 `$browser`

用途：

- 打开和录制本地 Web Demo。
- 操作普通网页。
- 截图、检查 DOM、验证交互。
- 录制网页操作流程。

优先用于：

```text
localhost
文档站
Web App
产品 Demo 页面
无需用户登录态的网页
```

原则：

- 默认后台操作。
- 需要用户观看时再显示浏览器。
- 不处理登录态依赖场景，登录态优先交给 `$chrome`。

### 3.2 `$chrome`

用途：

- 使用用户已有 Chrome 登录态。
- 操作 YouTube Studio。
- 操作需要账号状态的网站。

限制：

- 不读取 cookie、localStorage、密码、会话存储。
- 不默认创建账号、发布视频、上传文件。
- 上传和发布必须动作前确认。

### 3.3 `$computer-use`

用途：

- 操作本地 Mac GUI。
- 使用 OBS、剪辑软件、浏览器原生文件选择器。
- 处理没有 CLI/API 的软件。

限制：

- 删除本地文件、修改系统设置、上传文件、安装软件都要确认。
- 遇到密码、验证码、敏感数据输入必须交给用户或确认。
- 能用 CLI/API 做的，不优先用 GUI。

## 4. Agent 分工

这个项目里的 agent 不是单独跑多个模型，而是工程上定义清楚的职责模块。每个 agent 都有输入、输出和边界。

### 4.1 Brief Agent

职责：

- 理解用户需求。
- 补全必要假设。
- 识别平台、受众、时长、风格。
- 输出 `brief.yml`。

不能做：

- 不确认就开始录屏。
- 不确认就上传或发布。

### 4.2 Director Agent

职责：

- 生成视频方案。
- 拆章节、镜头、素材清单。
- 输出 `storyboard.yml` 和 `asset_plan.yml`。

不能做：

- 写无法录制的镜头。
- 为了好看增加无关内容。

### 4.3 Script Agent

职责：

- 生成旁白稿。
- 生成字幕初稿。
- 生成屏幕操作提示词。

质量标准：

- 短句。
- 旁白和画面对应。
- 技术名词不乱翻译。
- 命令和参数保留原样。

### 4.4 Capture Agent

职责：

- 调用 `$browser`、`$chrome`、`$computer-use` 采集素材。
- 记录每段素材来源。
- 输出 `assets/manifest.yml`。

确认点：

- 上传文件。
- 登录账号。
- 使用 Chrome 登录态。
- 发布、评论、提交表单。
- 修改本地或云端数据。

### 4.5 Voice Agent

职责：

- 根据旁白稿生成 AI 配音。
- 管理音色、语速、音量。
- 输出 `narration.mp3` 和 `voice_profile.yml`。

当前默认：

```yaml
provider: "edge-tts"
voice: "zh-CN-XiaoxiaoNeural"
```

### 4.6 Subtitle Agent

职责：

- 根据旁白稿生成字幕。
- 或用 Whisper 转写真实录音。
- 输出 `captions.srt` 和 `captions.vtt`。

规则：

- 已知旁白稿时，不优先用 Whisper 猜。
- Whisper 结果必须校对。

### 4.7 Editor Agent

职责：

- 组合素材、音频、字幕、标题卡。
- 生成横屏/竖屏版本。
- 调用 Auto-Editor 剪停顿。
- 输出 `draft.mp4` 和 `final.mp4`。

### 4.8 QA Agent

职责：

- 用 ffprobe 检查视频流、音频流、时长和编码。
- 抽关键帧检查黑屏和字幕。
- 检查字幕是否为空。
- 输出 `qa/report.md`。

### 4.9 Publisher Agent

职责：

- 生成标题、简介、标签和封面建议。
- 准备 YouTube 上传包。
- 经确认后调用 `$chrome` 或 YouTube API 上传。

禁止：

- 未确认就上传。
- 未确认就设置公开视频。
- 未确认就点击发布。

## 5. Harness 设计

### 5.1 运行目录

每次视频任务创建一个运行目录：

```text
runs/<run_id>/
  brief.yml
  approvals.yml
  state.json
  plan/
    storyboard.yml
    asset_plan.yml
  script/
    narration.zh.txt
    captions.srt
  assets/
    manifest.yml
  audio/
    narration.mp3
    voice_profile.yml
  render/
    draft.mp4
    final_16x9.mp4
    final_9x16.mp4
  qa/
    report.md
    screenshots/
  package/
    video.mp4
    thumbnail.png
    title.txt
    description.md
    tags.txt
```

### 5.2 状态机

```text
created
-> brief_ready
-> brief_approved
-> plan_ready
-> plan_approved
-> execution_ready
-> execution_approved
-> capturing
-> rendering
-> qa_ready
-> package_ready
-> upload_waiting_confirmation
-> uploaded
```

### 5.3 approvals.yml

所有确认都要落盘：

```yaml
brief:
  status: "approved"
  approved_at: "2026-06-08T00:00:00Z"
plan:
  status: "approved"
execution:
  status: "approved"
upload:
  status: "pending"
  reason: "Upload to YouTube requires explicit confirmation."
```

### 5.4 state.json

保存断点续跑信息：

```json
{
  "run_id": "20260608-001",
  "status": "package_ready",
  "current_stage": "package",
  "artifacts": {
    "brief": "brief.yml",
    "storyboard": "plan/storyboard.yml",
    "final_video": "render/final_16x9.mp4"
  }
}
```

## 6. CLI 规划

```bash
ai-video-maker new --type product_demo
ai-video-maker brief --run runs/20260608-001
ai-video-maker plan --run runs/20260608-001
ai-video-maker approve --run runs/20260608-001 --gate brief
ai-video-maker capture --run runs/20260608-001
ai-video-maker voice --run runs/20260608-001
ai-video-maker subtitle --run runs/20260608-001
ai-video-maker render --run runs/20260608-001
ai-video-maker qa --run runs/20260608-001
ai-video-maker package --run runs/20260608-001
ai-video-maker upload --run runs/20260608-001
```

`upload` 命令必须检查 `approvals.yml`，没有确认就拒绝执行。

## 7. 模板规划

### 7.1 通用 brief 模板

```yaml
goal: ""
audience: ""
platform: "youtube"
duration: 180
language: "zh-CN"
style: "技术讲解，AI 配音，字幕清晰"
source_material: []
must_show: []
must_avoid: []
upload:
  enabled: false
```

### 7.2 场景模板

| 模板 | 用途 |
|---|---|
| `general_demo` | 通用演示说明 |
| `repo_explainer` | 仓库/工具介绍 |
| `product_demo` | 产品功能演示 |
| `sop_tutorial` | 操作教程 |
| `bug_repro` | Bug 复现说明 |
| `api_tutorial` | API/SDK 教程 |
| `release_note` | 版本更新说明 |
