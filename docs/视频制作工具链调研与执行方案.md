# 视频制作工具链调研与执行方案

更新时间：2026-06-08

## 1. 目标

要做的事情不是 AI 漫剧，也不是纯素材混剪，而是技术类视频生产：

- 输入：一个开源仓库、本地项目、网页 Demo、终端实操流程。
- 输出：仓库介绍视频、实操演示视频、AI 配音、字幕、片头/章节卡、最终 MP4。
- 上传：优先自动上传 YouTube；如果 API 受限，再用 Chrome/YouTube Studio 手工流自动化辅助。
- 成本：优先免费、开源、可本地运行；必要时使用免费在线服务，但要标注风险。

核心判断：技术视频最适合用“脚本分析 + 录屏 + TTS 配音 + 自动字幕 + FFmpeg/MoviePy 合成”的流水线。剪映/CapCut 适合快速出片，但自动化程度和可控性不如命令行工具。

## 3. 推荐免费工具链

### 3.1 仓库分析与脚本生成

| 工具 | 用途 | 免费情况 | 老王判断 |
|---|---|---|---|
| `git` | 拉取仓库、查看提交、分支、README | 免费 | 必备 |
| `rg` | 快速搜索 README、入口、配置、API | 免费 | 比 grep 快，别犯懒 |
| `tree` / `find` | 生成项目结构 | 免费 | 用于视频里展示结构 |
| 项目自带测试/启动命令 | 验证项目能跑 | 免费 | 先跑通再录屏，别录翻车现场 |

产出：

- 视频大纲：项目是什么、解决什么问题、怎么安装、怎么跑。
- 实操脚本：终端命令、网页点击路径、关键功能演示。
- 旁白稿：给 TTS 使用。

### 3.2 浏览器和网页实操录制

| 工具 | 用途 | 免费情况 | 适合场景 |
|---|---|---|---|
| `$browser` + Playwright | 自动打开本地 Demo、点击、截图、录制网页视频 | 当前环境可用 | Web 项目、文档站、控制台演示 |
| `$chrome` | 使用用户已有 Chrome 登录态 | 当前环境可用 | YouTube Studio、需要登录的网站 |
| `$computer-use` | 操作本地 Mac GUI | 当前环境可用 | OBS、Kdenlive、浏览器 UI 兜底 |
| OBS Studio | 专业录屏、录窗口、录音频源 | 免费开源 | 桌面软件、复杂多窗口演示 |

Playwright 官方支持在浏览器上下文中录制视频；OBS 是免费开源录屏/直播工具。技术仓库介绍优先用 `$browser`/Playwright，因为它可复现、可脚本化、少出幺蛾子。只有录桌面软件或必须走 GUI 时，才用 OBS 或 `$computer-use`。

### 3.3 终端实操录制

| 工具 | 用途 | 免费情况 | 说明 |
|---|---|---|---|
| VHS | 把终端命令脚本录成 GIF/MP4 风格素材 | 开源免费 | 适合展示安装、运行、CLI 操作 |
| asciinema | 录制终端会话 | 开源免费 | 适合保留命令过程 |
| FFmpeg | 录屏、裁剪、合成终端素材 | 开源免费 | 最终还是靠它收尾 |

技术视频里“终端 + 浏览器”是主画面，不要搞一堆无意义素材堆砌。仓库讲解靠清晰，不靠花活。

### 3.4 AI 配音

| 工具 | 类型 | 免费情况 | 优点 | 风险/缺点 |
|---|---|---|---|---|
| `edge-tts` | 在线 TTS CLI | 免费调用 Microsoft Edge 在线 TTS 服务 | 声音自然，可生成音频和字幕 | 不是官方独立 API，稳定性和条款要留意 |
| Kokoro | 本地/开权重 TTS | Apache-2.0 | 轻量，适合免费本地化 | 中文音色和效果需实测 |
| Piper | 本地 TTS | 免费开源 | 可离线 | 原仓库已归档，作为备用 |
| 剪映/CapCut 文本朗读 | 平台内置 TTS | 基础能力免费 | 抖音风格强，上手快 | 自动化弱，依赖平台功能 |

推荐顺序：

1. 想要“好听、快出片”：先用 `edge-tts`。
2. 想要“本地免费、少依赖平台”：试 Kokoro。
3. 想要“剪映同款短视频味”：剪映/CapCut 内置文本朗读。

### 3.5 字幕与转写

| 工具 | 用途 | 免费情况 | 说明 |
|---|---|---|---|
| Whisper | 语音转文字、生成字幕 | MIT 开源 | 适合从旁白或实录音频生成 SRT |
| `edge-tts` | 生成 TTS 音频时同步输出字幕 | 免费 | 适合旁白稿固定的场景 |
| FFmpeg | 烧录字幕、调整字幕样式 | 开源免费 | 最终合成必备 |
| Subtitle Edit | 人工校对字幕 | 开源免费 | GUI 兜底 |

对于技术视频，最稳的是先写旁白稿，再 TTS 生成音频和字幕。不要先录一堆口水音再转写，后期会被字幕校对折磨。

### 3.6 程序化剪辑与视频合成

| 工具 | 用途 | 免费情况 | 说明 |
|---|---|---|---|
| FFmpeg | 拼接、裁剪、转码、字幕、混音 | 开源免费 | 视频流水线核心 |
| MoviePy | Python 自动剪辑、加图层、加标题 | MIT 开源 | 适合仓库讲解视频脚本化生产 |
| Auto-Editor | 自动剪静音、停顿、无动作片段 | CLI 开源免费 | 可缩短录屏素材 |
| Remotion | React 写视频动效、章节卡、片头 | 个人/小团队免费；公司需看授权 | 适合做高级标题卡 |
| Kdenlive / Shotcut / OpenShot | GUI 剪辑兜底 | 免费开源 | 最后人工微调用 |

最小可落地组合：

```text
FFmpeg + MoviePy + edge-tts + Whisper + Playwright + Auto-Editor
```

这套能覆盖：录屏、配音、字幕、剪静音、合成、导出 MP4。

### 3.7 YouTube 上传

| 工具 | 用途 | 免费情况 | 风险 |
|---|---|---|---|
| YouTube Data API `videos.insert` | 自动上传视频和元数据 | 免费额度可用 | 新 API 项目未审查时，上传视频可能被限制为 private |
| `youtubeuploader` | 基于 YouTube API 的 CLI 上传器 | 开源免费 | 仍依赖 OAuth 和 API 限制 |
| `$chrome` + YouTube Studio | 用用户已有登录态上传 | 免费 | 上传/发布是外部副作用，执行前必须确认 |

YouTube 官方文档说明 `videos.insert` 可上传视频，但 2020-07-28 之后创建的未验证 API 项目上传内容会限制为 private，直到项目通过审核。实际执行时建议：

1. 有已审核 API 项目：走 YouTube Data API。
2. 没有已审核 API 项目：先导出成片，再用 `$chrome` 打开 YouTube Studio 辅助上传。
3. 到“上传文件、设置公开、发布”这些动作前，必须让用户确认。这个不能瞎点，点错了就成事故。

## 4. 抖音技术视频博主通常用什么

### 4.1 先说结论

不能凭一个抖音成片就断言某个博主一定用了哪套工具。除非博主在评论区、视频简介、直播课或教程里公开说了工具栈，否则只能做合理判断。

但从技术类短视频常见形态看，主流不是开源 CLI 流水线，而是：

```text
剪映/CapCut + AI 配音/文本朗读 + 自动字幕 + 录屏工具 + 模板化剪辑
```

更专业或更商业化的账号，可能会用：

```text
OBS / ScreenFlow / Camtasia / Premiere Pro / DaVinci Resolve
```

技术讲解类视频和 AI 漫剧不是一类东西。技术视频主要是：

- 屏幕录制：IDE、终端、浏览器、GitHub、Demo 页面。
- AI 旁白：解释项目是什么、怎么跑、效果如何。
- 自动字幕：大字字幕、关键词高亮。
- 节奏剪辑：删停顿、放大重点区域、加箭头/框选。
- 封面标题：强标题、技术关键词、结果预览。

AI 漫剧那种文生图、图生视频、角色一致性、分镜生成，不是这里的主线。

### 4.2 他们用的是不是上面这些开源工具

大概率不是完全一样。

抖音上很多技术号更可能用剪映/CapCut，因为它省事：

- 自带文本朗读。
- 自带自动字幕。
- 自带模板、转场、音效。
- 竖屏 9:16 适配方便。
- 和抖音发布链路贴得近。

但它们背后的功能模块，和我们方案里的工具是同一类能力：

| 抖音/剪映常见能力 | 我们的免费自动化替代 |
|---|---|
| 文本朗读 / AI 配音 | `edge-tts`、Kokoro、Piper |
| 自动字幕 | Whisper、`edge-tts` 字幕、FFmpeg 字幕烧录 |
| 录屏 | Playwright、OBS、VHS、asciinema |
| 剪静音/提节奏 | Auto-Editor、MoviePy |
| 模板标题/章节卡 | Remotion、MoviePy、FFmpeg |
| 竖屏导出 | FFmpeg、MoviePy |
| 平台上传 | YouTube Data API、`youtubeuploader`、`$chrome` |

所以准确说法是：抖音技术类博主常用的是“同类功能”，不一定是同名工具。我们采用开源/免费工具，是为了把这套生产方式变成可复现、可批量、可自动化的流程。

### 4.3 为什么不直接全用剪映

剪映适合人手剪一条视频，但不适合让我稳定替你批量做仓库讲解：

- 自动化弱：很难完整脚本化“拉仓库、跑 demo、录屏、配音、剪辑、导出”。
- 依赖平台：功能、音色、授权、导出限制可能变化。
- 可复现差：同一条视频以后改稿、换仓库、换配音，不如代码流水线稳定。
- 不利于批量：做 1 条可以，做 20 条就开始痛苦。

更务实的策略：

- 批量自动化主线：FFmpeg / MoviePy / Playwright / TTS / Whisper。
- 快速抖音风格兜底：剪映/CapCut。
- 专业人工微调：Kdenlive / DaVinci Resolve / Premiere。

## 5. 推荐执行流程

### 5.1 仓库介绍视频流水线

```text
1. 读取仓库
   - README
   - package/requirements/build 配置
   - 入口文件
   - 示例和文档

2. 跑通项目
   - 安装依赖
   - 启动服务
   - 跑测试或示例
   - 记录可演示功能

3. 生成视频脚本
   - 开场 10 秒：这个项目解决什么问题
   - 中段：安装、核心结构、实操演示
   - 结尾：适合谁用、注意点、下一步

4. 录制素材
   - 浏览器 Demo：Playwright / $browser
   - 终端操作：VHS / asciinema
   - 桌面操作：OBS / $computer-use

5. 生成配音和字幕
   - 首选 edge-tts
   - 本地化可试 Kokoro
   - 字幕用 TTS 输出或 Whisper 校正

6. 自动剪辑
   - Auto-Editor 剪静音
   - MoviePy 组合片段
   - FFmpeg 混音、转码、烧字幕

7. 导出
   - YouTube 横屏：1920x1080
   - 抖音/Shorts 竖屏：1080x1920
   - 必要时导出两个版本

8. 上传
   - 优先 YouTube Data API
   - API 受限则 $chrome 辅助 YouTube Studio
```

### 5.2 视频结构模板

```text
00:00 - 00:10  项目一句话价值
00:10 - 00:30  效果预览
00:30 - 01:00  仓库结构和技术栈
01:00 - 02:30  安装与启动
02:30 - 04:30  核心功能实操
04:30 - 05:10  代码亮点或架构解释
05:10 - 05:40  注意事项、适用场景、总结
```

短视频版本可以压缩成：

```text
00:00 - 00:03  强钩子：这个开源项目能干什么
00:03 - 00:15  效果展示
00:15 - 00:45  三步跑起来
00:45 - 01:10  核心功能演示
01:10 - 01:20  总结和仓库名
```

## 6. 初始安装建议

后续真正执行前，再按需安装，不要一口气装一堆没用的东西。

```bash
rtk pipx install edge-tts
rtk pipx install auto-editor
rtk pipx install openai-whisper
rtk pipx install moviepy
rtk npm create video@latest
rtk brew install --cask obs
```

注意：安装软件、上传文件、发布视频这类动作，执行前需要单独确认。老王嘴臭归嘴臭，这种会影响账号和系统的操作不能瞎来。

## 7. 自动化方案优先级

### 方案 A：最推荐，自动化优先

```text
Playwright/$browser + edge-tts + Whisper + MoviePy + FFmpeg + Auto-Editor
```

优点：

- 免费。
- 自动化强。
- 可以重复生成。
- 适合批量做开源仓库介绍。

缺点：

- 初次搭流水线要调参数。
- 复杂视觉包装不如剪映模板快。

### 方案 B：抖音风格快速出片

```text
录屏 + 剪映/CapCut + 文本朗读 + 自动字幕
```

优点：

- 快。
- 抖音味强。
- 人工微调方便。

缺点：

- 自动化弱。
- 长期批量做会累。
- 依赖平台功能。

### 方案 C：专业剪辑兜底

```text
OBS + Kdenlive / DaVinci Resolve / Premiere + TTS + 字幕
```

优点：

- 控制力强。
- 复杂视频效果更好。

缺点：

- 人工成本高。
- 不适合每个仓库都从头手剪。

## 8. 下一步落地建议

拿一个具体仓库做试点，不要空谈。最小验证目标：

```text
1. 生成 60-90 秒仓库介绍短视频。
2. 包含浏览器或终端实操录屏。
3. 包含 AI 配音。
4. 包含同步字幕。
5. 导出 1080p MP4。
6. 给出 YouTube 标题、简介、标签草稿。
```

试点跑通后，再扩展：

- 横屏 YouTube 版。
- 竖屏 Shorts/抖音版。
- 自动封面。
- 批量处理多个 GitHub 仓库。
- 自动上传 YouTube。
